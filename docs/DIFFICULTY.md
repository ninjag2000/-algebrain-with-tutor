# Как определяется сложность заданий

## 1. Уровень сложности (тир)

- **Тип:** `DifficultyLevel = 1 | 2 | 3 | 4 | 5 | 6 | 7` (см. `types.ts`).
- **Смысл:** тир 1 — самый лёгкий, тир 7 — самый сложный.
- **Структура данных:** у каждого навыка (арифметика, уравнения, степени и логарифмы, тригонометрия) есть **7 тиров** — массив массивов шаблонов: `tiers[skill][0]` … `tiers[skill][6]` соответствуют тирам 1–7.
- **Откуда берётся сложность у задания:** при генерации вопроса вызывается `generateOneQuestion(seed, difficulty, skill)`. Берётся массив шаблонов **по индексу `difficulty - 1`** (т.е. тир = выбранный уровень сложности). Из шаблона получают `question`, `answer`, `acceptedAnswers`; поле **`difficulty` в возвращаемом вопросе всегда подменяется на переданный уровень** (строка 377 в `mockData.ts`: `return { ...q, difficulty }`). То есть **сложность задания = номер тира (1–7), в котором оно лежит**, а не значение `difficulty` внутри шаблона.

Итог: **сложность задания определяется тем, в каком тире (1–7) оно лежит в `mockData.ts`** для данного навыка.

---

## 2. Как подбирается сложность для пользователя

Сложность задаётся **выбором в онбоардинге** (экран персональности). Функция `getBaseDifficulty(preparationLevel)` возвращает один тир для всех навыков:

| Выбор в онбоардинге | Сложность (тир) |
|---------------------|-----------------|
| **Middle School** или не выбран | **2** (базовая) |
| **High School** | **3** |
| **Advanced Math** | **4** |

Уровень подготовки **не сохраняется** после прохождения дня и не меняется от точности/серии — используется только выбор пользователя при онбоардинге (`personalization.level`). При генерации практики и дневного челленджа для всех заданий берётся один и тот же тир (2, 3 или 4) по всем навыкам.

---

## 3. Fallback при пустом тире

Если для выбранного навыка и тира массив шаблонов **пустой** (`tierTemplates.length === 0`), вопрос берётся из списка `mockQuestions` с тем же навыком (по `getSkillFromQuestion`), а сложность у этого вопроса принудительно ставится равной запрошенному `difficulty` (тиру). То есть при пустом тире сложность по-прежнему считается по номеру тира (1–7), а не по полю `difficulty` у fallback-вопроса в `mockQuestions`.

---

## 4. Где что лежит в коде

- **Типы:** `types.ts` — `DifficultyLevel`, `PracticeQuestion.difficulty`.
- **Тиры и шаблоны:** `mockData.ts` — `buildTiersBySkill()`, массивы по навыкам (например, `logarithms()`), комментарии «Тир 1» … «Тир 7».
- **Генерация одного вопроса:** `mockData.ts` — `generateOneQuestion(seed, difficulty, skill)` — выбор тира по `difficulty`, подстановка `difficulty` в ответ.
- **Базовая сложность по онбоардингу:** `mockData.ts` — `getBaseDifficulty(preparationLevel)` возвращает 2 (middle/null), 3 (high = High School), 4 (advanced). Тип `PreparationLevel` = `'middle' | 'high' | 'advanced' | null`.
- **Пары для практики:** `getPracticeDifficultySkillPairs(count, baseDifficulty)` — одна сложность (2, 3 или 4) для всех заданий, навыки по кругу.
- **Список уровней 1–7:** `mockData.ts` — `DIFFICULTY_LEVELS`.
