# Проверка, что на Android загружается новая вёрстка калькулятора

## Предупреждение flatDir

Сообщение `Using flatDir should be avoided...` при сборке Android — известное предупреждение: `flatDir` используется для локальных AAR Capacitor/Cordova плагинов. На сборку и работу приложения оно не влияет, его можно игнорировать.

## Убедиться, что подхватилась новая сборка

1. **Сборка и синк**
   ```bash
   npm run build
   npx cap sync android
   ```

2. **Проверка бандла (опционально)**  
   В папке `dist/assets/` после `npm run build` должен быть обновлённый JS (по дате файла). В нём можно поискать строку `data-exponent-render` — она есть только в новой вёрстке.

3. **Проверка на устройстве через Chrome**
   - Подключите телефон по USB, включите отладку по USB.
   - На ПК откройте Chrome и перейдите: `chrome://inspect/#devices`.
   - Найдите WebView вашего приложения и нажмите **inspect**.
   - В калькуляторе введите `2^(5)`.
   - В инспекторе найдите контейнер поля ввода (например, div с классом `min-h-[1.5rem]`). У него должен быть атрибут **`data-exponent-render="1"`**, если загружена новая версия.

   - Если атрибута **нет** — на устройстве всё ещё старый бандл: переустановите приложение или очистите данные приложения и установите заново из Android Studio.
   - Если атрибут **есть**, а степень всё равно отображается как `2^(5)` — значит, нужно смотреть логику рендера в коде.

4. **Жёсткое обновление на устройстве**
   - Удалите приложение с телефона.
   - В Android Studio: **Build → Clean Project**, затем **Build → Rebuild Project**.
   - Установите приложение заново (Run).
